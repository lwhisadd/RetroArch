# 討論紀錄 01 - 遊戲清單（.lpl）新增下載功能實作計劃

本計劃旨在為 RetroArch 新增一項功能：當使用者在遊戲清單（.lpl）的操作選單（子選單）中點選某個遊戲時，若該遊戲符合特定條件，則可看到「下載」選項，點選後將在背景自動下載遊戲 ROM 至對應的路徑下。

此專案已完成所有的需求確認，以下為最終的技術實作計劃。

---

## 📋 需求確認總結
1.  **資料格式**：在 `.lpl` 的 JSON 檔案中，各個遊戲項目（`items` 陣列）包含 `"download"` 欄位，例如 `"download": "https://example.com/download/mario.zip"`。
2.  **呈現位置**：與「執行」、「刪除」等選項並列在選取該遊戲後的「操作選單（子選單）」中。
3.  **儲存路徑**：下載後直接儲存至項目中 `"path"` 欄位指定的本地路徑（例如：`D:\Roms\NES\mario.nes`），若該路徑上的實體檔案已存在，則不允許重複下載。
4.  **下載機制**：接入 RetroArch 內建的異步背景 HTTP 傳輸任務系統（HTTP Transfer Task），顯示下載進度條並在完成時彈出通知。
5.  **顯示與隱藏條件**：
    *   若無 `"download"` 欄位或為空：**隱藏**「下載」選項。
    *   若 `"path"` 指定的本地檔案已存在：**隱藏**「下載」選項。
6.  **目錄處理**：若 `"path"` 的上層資料夾目錄（如 `D:\Roms\NES\`）在硬碟中還不存在，系統將自動遞迴建立父資料夾，再寫入檔案。
7.  **選單更新**：下載成功後不主動重新整理選單，直到使用者手動回到上一頁再進入才會刷新選項狀態。
8.  **單一下載限制**：同一時間僅允許進行一個下載任務，若已有下載任務進行中，則提示並拒絕新的下載。

---

## 🛠️ 技術實作方案

本功能主要涉及 RetroArch 的清單解析、UI 呈現、背景任務與檔案操作等模組。以下為模組化設計：

### 1. 資料結構擴充 (`playlist.h`)
在播放列表項目的結構體 `struct playlist_entry` 中新增存放下載網址的指標：
```c
struct playlist_entry
{
   char *path;
   char *label;
   ...
   char *download; /* 新增：存放下載網址 */
   ...
};
```

---

### 2. 播放列表解析與寫入 (`playlist.c`)
*   **JSON 欄位解析**：
    在 `JSONObjectMemberHandler` 的欄位解析邏輯（`case 'd'`）中，新增對 `"download"` 欄位的讀取：
    ```c
    case 'd':
       if (!strcmp(pValue, "db_name"))
          pCtx->current_string_val = &pCtx->current_entry->db_name;
       else if (!strcmp(pValue, "download"))
          pCtx->current_string_val = &pCtx->current_entry->download; /* 新增 */
       break;
    ```
*   **記憶體釋放**：
    在 `playlist_free_entry` 釋放結構體時，確保安全釋放 `download` 記憶體：
    ```c
    if (entry->download)
       free(entry->download);
    entry->download = NULL;
    ```
*   **JSON 檔案寫入**：
    在 `playlist_write_file` 中，如果 `download` 指標不為空，在儲存清單時也將其寫回 JSON 檔案，避免資訊丟失：
    ```c
    if (playlist->entries[i].download)
    {
       rjsonwriter_raw(writer, ",\n", 2);
       rjsonwriter_add_spaces(writer, 6);
       rjsonwriter_add_string(writer, "download");
       rjsonwriter_raw(writer, ": ", 2);
       rjsonwriter_add_string(writer, playlist->entries[i].download);
    }
    ```

---

### 3. 多國語系語系定義 (`msg_hash.h`, `intl/*`)
*   定義選單與通知的語系常數：
    *   `MENU_ENUM_LABEL_PLAYLIST_ENTRY_DOWNLOAD`
    *   `MENU_ENUM_LABEL_VALUE_PLAYLIST_ENTRY_DOWNLOAD` (預設為 `"Download"`)
*   在 `intl/msg_hash_zh_tw.h` 中，定義繁體中文顯示值：
    *   `"下載遊戲"`

---

### 4. 菜單顯示與隱藏邏輯 (`menu/menu_displaylist.c`)
在 `menu_displaylist_parse_horizontal_content_actions` 函數中（用於建立單一遊戲項目的子操作選單）：
*   **前置條件檢查**：
    1.  檢查該遊戲項目是否具備下載連結：`bool has_download = (entry && entry->download && *entry->download);`
    2.  檢查本地路徑檔案是否已存在：`bool file_exists = path_is_valid(entry->path);`
*   **加入選單選項**：
    若 `has_download` 為真，且 `file_exists` 為假，則將下載選項加入選單：
    ```c
    if (has_download && !file_exists)
    {
       menu_entries_append(list,
             msg_hash_to_str(MENU_ENUM_LABEL_VALUE_PLAYLIST_ENTRY_DOWNLOAD),
             msg_hash_to_str(MENU_ENUM_LABEL_PLAYLIST_ENTRY_DOWNLOAD),
             MENU_ENUM_LABEL_PLAYLIST_ENTRY_DOWNLOAD,
             FILE_TYPE_PLAYLIST_ENTRY, 0, idx, NULL);
    }
    ```

---

### 5. 點擊按鈕回呼與下載流程 (`menu/cbs/menu_cbs_ok.c`)
*   **單一下載鎖定義**：
    定義一個全局靜態狀態變數：
    ```c
    static bool g_playlist_download_in_progress = false;
    ```
*   **實作下載動作回呼** `action_ok_playlist_entry_download()`：
    1.  **併發檢查**：檢查 `g_playlist_download_in_progress` 是否為 `true`。若是，呼叫 `runloop_msg_queue_push` 彈出警告訊息「已有下載任務進行中，請稍候」，並返回。
    2.  **上鎖**：設定 `g_playlist_download_in_progress = true;`。
    3.  **遞迴目錄建立**：
        使用 `fill_pathname_parent_dir` 解析出目標實體目錄（例如從 `D:\Roms\NES\mario.nes` 得到 `D:\Roms\NES`）。
        若該目錄不存在（`!path_is_directory(parent_dir)`），則呼叫內建的 `path_mkdir(parent_dir)` 進行遞迴目錄建立。
    4.  **初始化傳輸任務**：
        配置 `file_transfer_t` 結構體，將下載 URL 複製至該結構體，並將目標儲存路徑（`entry->path`）設定至 `file_transfer_t.path` 中。
    5.  **啟動背景傳輸任務**：
        呼叫 `task_push_http_transfer_file()` 將任務推入背景隊列，並註冊專用的回呼函數（例如 `playlist_entry_download_callback`）。
*   **任務完成回呼** `playlist_entry_download_callback()`：
    無論下載成功或失敗，在任務最終結束時解鎖：
    ```c
    g_playlist_download_in_progress = false;
    ```
    系統會自動透過內建的 OSD 機制通知使用者下載結果。

## 🧪 驗證與測試計畫

### 1. 單元測試與功能驗證
*   **格式解析測試**：
    準備一個包含 `"download"` 欄位的 `.lpl` 檔案，載入後確認 `struct playlist_entry` 中的 `download` 指標能正確獲取對應網址。
*   **顯示與隱藏條件測試**：
    *   測試 1：檔案不存在且包含 `download` 欄位 -> 確認選單有出現「下載遊戲」選項。
    *   測試 2：檔案不存在但無 `download` 欄位 -> 確認無「下載遊戲」選項。
    *   測試 3：檔案已存在且包含 `download` 欄位 -> 確認無「下載遊戲」選項。
*   **目錄自動建立測試**：
    在硬碟中刪除該遊戲設定路徑的上層目錄，點擊「下載遊戲」後，驗證系統是否能成功建立該目錄並正確寫入檔案。
*   **單一下載併發測試**：
    在下載遊戲 A 期間，快速切換至遊戲 B 的選單，確認其不會重複顯示或點擊時能被系統攔截並提示「已有下載任務進行中」。
